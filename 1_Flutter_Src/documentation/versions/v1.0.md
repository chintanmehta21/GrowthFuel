# GrowthFuel v1.0 - End-to-End App Documentation

## Overview
GrowthFuel is a Flutter-based fitness and habit tracking app integrated with Supabase for backend services. It enables users to log workouts, track habits, view historical analytics, and receive personalized insights. The app is optimized for Android and uses Supabase for authentication, data storage, and real-time updates.

---

## Project Structure

- **Flutter App Directory:**
  - `lib/`
    - `main.dart`: App entry point, theme, routing, Supabase initialization.
    - `config/`: Theme and Supabase config files.
    - `models/`: Data models for user, exercise, habit, stats.
    - `services/`: Service layer for Supabase, authentication, CRUD operations.
    - `screens/`: UI screens for splash, auth, home, exercise, habits, history, profile.
    - `widgets/`: Reusable UI components (score cards, navigation, charts).
    - `utils/`: Constants and date utilities.

---

## Supabase Backend

- **Authentication:**
  - Email/password and Google OAuth2.0 sign-in.
  - Auto-profile creation on signup.
- **Database Schema:**
  - `profiles`: User profile data.
  - `exercise_names`: Master list of exercises.
  - `exercises`: User workout data (with sets).
  - `habit_names`: Master list of habits.
  - `habits_data`: Daily habit tracking.
  - `daily_stats`: Calculated daily statistics.
- **Row Level Security (RLS):**
  - Users can only access their own data.
- **Triggers & Functions:**
  - Daily stats calculation and habit data upload (via SQL functions/triggers).

## App Features

### Authentication & Onboarding
- Splash screen with branding.
- Login and signup screens (Supabase auth).
- Google Sign-In integration.
- Auto-profile creation on signup.

### Home Screen
- Personalized greeting with user name.
- Workout score card and habit score card widgets.
- Weekly workout count display.
- Calories burned chart (fl_chart).
- Real-time stats from Supabase.

### Exercise Tab
- Tab bar with 5 categories: Push, Pull, Legs, Upper, Lower.
- Exercise list UI for each category.
- Exercise input form (sets, previous/current weight, reps).
- "+ ADD SET" and "+ ADD EXERCISE" functionality.
- "FINISH" button saves data to Supabase.
- Data persistence and retrieval from Supabase.

### Habits Tab
- Habits list view.
- "Add New Habit" modal/screen.
- Habit types: Durational (timer), Read (page counter), Task (checkbox), Count (counter).
- Habit card widgets for each type.
- Daily reset logic (00:01).
- Habit completion tracking.

### History Tab
- Tab switcher: Daily, Weekly, Monthly views.
- Daily view: workout/habit cards for selected date.
- Weekly view: summary of workouts/habits.
- Monthly view: calendar with indicators.
- Historical data fetch from Supabase.
- Date navigation (previous/next).

### Profile Tab
- Profile screen layout.
- Display user information.
- Settings: Dark Mode, Units of Measure.
- Support links: Help Center, Privacy Policy.
- Logout functionality.
- Connects to user profile in Supabase.

### Bottom Navigation
- Bottom navigation bar with 5 main tabs.
- Navigation between screens.
- Styled with accent colors.

### Database Integration & API
- Supabase service layer for all CRUD operations.
- Daily stats API calls and sync logic.
- Database triggers for end-of-day habit upload.
- Scheduled tasks for daily API calls.

### Testing & Polish
- Unit, widget, and integration tests.
- Manual verification of Supabase setup, app flows, UI, and performance.

## How Data Flows
- User authenticates via Supabase.
- Data is stored in Supabase tables with RLS for privacy.
- Stats are calculated via triggers/functions and displayed in the app.
- All CRUD operations are handled via service layer.
